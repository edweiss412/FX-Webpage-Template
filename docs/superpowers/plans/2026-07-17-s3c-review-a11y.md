> **For agentic workers:** TDD per task (invariant 1): failing test → minimal impl → passing test → commit. Steps use `- [ ]`.

**Goal:** resolve two live P2 a11y findings on the consolidated Step-3 review surface — S3C-1 (color-only status dots) and S3C-2 (modal background not `inert`) — and stale-mark S3C-3.

- [ ] **Step 3b: Implement effect.** In `Step3ReviewModal`, add a mount/unmount effect (empty deps), next to the scroll-lock effect:

```tsx
useEffect(() => {
  const roots = Array.from(document.querySelectorAll<HTMLElement>("[data-inert-root]"));
  const prev = roots.map((el) => ({ el, inert: el.inert, ariaHidden: el.getAttribute("aria-hidden") }));
  for (const el of roots) { el.inert = true; el.setAttribute("aria-hidden", "true"); }
  return () => {
    for (const { el, inert, ariaHidden } of prev) {
      el.inert = inert;
      if (ariaHidden === null) el.removeAttribute("aria-hidden");
      else el.setAttribute("aria-hidden", ariaHidden);
    }
  };
}, []);
```

(`HTMLElement.inert` is a boolean DOM prop — TS `lib.dom` includes it. If typecheck complains in the harness's TS config, it will not — `inert` is standard in the repo's `lib`.)
