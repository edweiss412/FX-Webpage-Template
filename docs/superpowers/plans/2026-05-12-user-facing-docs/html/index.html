FXAV turns Doug Larson’s per-show Google Sheets into mobile-friendly, per-crew-member webpages. The rest of the milestones built the machinery. M12 builds the manual — gated behind admin login, built like a tiny wiki, and stitched into the existing admin UI so the docs meet Doug where he’s already confused.

Most enterprise apps would write generic SaaS help docs and call it done. M12 doesn’t. Doug is the only reader. If a sentence wouldn’t help him, it doesn’t ship.

Ten plan-wide invariants — non-negotiable

M12 extends AGENTS.md’s nine global invariants with one M12-specific rule. Violating any of them is a P0 bug regardless of test status. The full list, with citations, lives in Overview.

  1. TDD per task.
  2. Commit per task — <type>(help|messages|time|screenshots): <summary>.
  3. Spec is canonical.
  4. No raw error codes in user-visible UI — /help/errors shows codes only as URL fragments.
  5. impeccable v3 UI gate on every /help/* page before close-out.
  6. Supabase call-boundary discipline — already pinned for requireAdmin().
  7. MessageCatalogEntry extension is additive.
  8. Catalog–master-spec alignment — admin-log-only codes have all six user-facing fields nulled.
  9. lib/time/now.ts is the only render-side clock — server-time grep guard enforces.
  10. §5.6 affordance matrix is the §9.0.1 retrofit contract.
