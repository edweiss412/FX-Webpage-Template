PHASE C

One clock. Pinnable on demand. Tamper-proof in prod.

For screenshots to never drift, the page must look the same every time you capture it. That means a relative timestamp like "updated 4 minutes ago" can’t silently tick to "5 minutes ago" between two runs. Phase C builds a server-side clock that the screenshot harness can freeze, and only the screenshot harness — production stays on real time.

Imagine you take a screenshot of the admin dashboard at exactly 3:00:00 PM. It says "last synced 4 minutes ago." Tomorrow, the harness runs again at 3:00:30. Same fixture data, same page — but now it says "5 minutes ago," because real time moved on. The screenshot bytes diff. CI fails. Forever.

Phase C solves it: a server-side clock that only the screenshot harness can freeze, via a special header and a Bearer secret. Production never gets to freeze the clock. The harness pins "now" to 2026-03-24T15:00:00.000Z every time, and the screenshot says "4 minutes ago" forever.

Without vs with the frozen-clock utility

The three-precondition gate

All three must be true at the same time for nowDate() to return the frozen instant. Any one missing → real time. Defense in depth: any single misconfiguration in production leaves at least one lock intact.

PRECONDITION 1

Env flag

ENABLE_TEST_AUTH === "true". A boot-time environment variable. Production never sets it. Requires deployment-level access to change.

PRECONDITION 2

Bearer secret

Authorization: Bearer ${TEST_AUTH_SECRET}. Shared secret only the harness knows. Constant-time compared. Missing → falls through.

PRECONDITION 3

Frozen-now header

X-Screenshot-Frozen-Now: <ISO 8601>. The actual frozen instant. Parsed via Date; rejected if invalid.

02 The waiver mechanism

Not every new Date() is render-side. An audit log timestamp needs the real wall-clock instant, even when called from a route the harness can reach. C.4 supports per-line waivers: 

// not-render-side: audit log timestamp, must be real wall-clock instant
await supabase.from("admin_audit_log").insert({
  event: "show_resync_clicked",
  at: new Date().toISOString(),
});

The grep guard parses each new Date() hit, looks back ≤3 lines for a not-render-side: comment, and waives it. No comment → must migrate to nowDate().
