The three tasks

I.1 · impeccable v3

/impeccable critique + audit

Both, not one or the other. Critique surfaces UX/accessibility/content concerns. Audit checks against v3 preflight gates (PRODUCT.md, DESIGN.md, design register). HIGH/CRITICAL findings: fix inline OR defer via DEFERRED.md. Re-run both until zero un-deferred high/critical.

I.2 · cross-CLI review

/codex:adversarial-review

Implementer is Claude Code; reviewer is Codex. Reviewer reads spec + plan + implementation, returns APPROVE or findings. Critical: --base <M12-base-SHA> + --scope branch so every round reviews the full M12 diff, not the latest fix. Iterate fix → review until APPROVE.

I.3 · handoff

handoffs/M12-help.md

Twelve sections: spec sections in scope, AC list with PASS/DEFERRED/N/A, amendments in scope, test commands + expected output, convergence log, watchpoints, deferred content, impeccable findings, perf/bundle impact, convergence-log transcript, routing decision, sign-off. Updates ROUTING.md with the M12 row. Runs final pnpm build against the post-content state.

Cross-CLI reviewer watchpoints

The reviewer anchors on these. Each is a documented bug class from prior milestones, codified here so they’re explicitly audited.

  • Catalog drift: every admin-log-only code has dougFacing: null in the live catalog.
  • Clock pipeline: every render-side new Date() uses nowDate() or carries a per-line waiver.
  • Affordance retrofit completeness: every concrete row in AFFORDANCE_MATRIX has its testid in the live component.
  • Preview-as-crew exception: previewed content emits no Learn more →.
  • /help/errors trailing CTA: "If this keeps happening, tell Eric →" — not a self-linking Learn more →.
  • <picture> contract: every WebP pair (light + dark) exists; <Screenshot> emits the dark <source>.
  • Invariant #4: /help/errors shows codes only as anchor fragments; visible heading is entry.title.
