PHASE D

Six building blocks. One vocabulary. Every page speaks it.

MDX is Markdown that can render React components. Phase D builds the six components every help page uses, then registers them in mdx-components.tsx so they’re available everywhere under app/help/ without per-page imports.

Every help page is mostly Markdown — headings, paragraphs, lists. But certain things repeat across pages: a yellow "heads up" box, a numbered procedure step, a screenshot, an anchored heading you can deep-link to. Rather than reinventing those for every page, Phase D builds them once as components, then makes them available in every .mdx file by name. Page authors write <Callout type="warning">; the component handles the icon, the color, the role, and the accessibility plumbing.

The vocabulary

D.1 · <Callout>

Boxed message · 3 variants

Three variants — note, warning, tip — with palette tokens from globals.css. Fixed icon per type. Unknown type falls back to note per spec §6.3 guard. Used for "heads up" guidance interspersed in body text.

_components/Callout.tsx
D.2 · <Step>

Numbered procedure item

Auto-incrementing via CSS counter — authors don’t manually number. Used in First-time setup and similar adoption flows.

_components/Step.tsx
D.3 · <ScreenshotPlaceholder>

Draft scaffold · lint-prohibited at v1

A labeled empty box for pages authored before Phase F’s real screenshots land. Phase H Task H.5 enforces zero references in shipped v1 MDX. Useful only during the gap.

_components/ScreenshotPlaceholder.tsx
D.4 · <Screenshot name>

Production · <picture> with light/dark

Renders <picture> with light/dark WebP sources from public/help/screenshots/<name>-{light,dark}.webp. Prop is name, not key — see the reserved-key trap below. Empty name throws.

_components/Screenshot.tsx
D.5 · <RefAnchor id>

Anchored heading · catalog-code shape

Heading with explicit id for deep-linking. Renders <h2> by default; as="h3" for nested anchors on /help/errors. id must match /^[A-Z][A-Z0-9_]*$/ (catalog-code shape). Includes click-to-copy link affordance.

_components/RefAnchor.tsx
D.6 · <TipFromSheets>

Bridge metaphor callout

Visually distinct from <Callout type="tip"> — amber palette, small from Sheets label so the anchoring metaphor stays obvious. Used only on the four adoption-track pages.

_components/TipFromSheets.tsx

02 The key/name trap · a round-1 finding
Why <Screenshot name> not <Screenshot key>

key is a React-reserved attribute. The reconciler consumes it before it reaches the component’s props object. The original draft used <Screenshot key="dashboard-active-shows"> and would have rendered /help/screenshots/undefined-light.webp every time. Rename to name. Add a regression test asserting the rendered HTML never contains the literal string "undefined".
