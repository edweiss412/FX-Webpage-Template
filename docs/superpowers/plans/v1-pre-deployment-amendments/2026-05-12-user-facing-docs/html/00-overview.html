01 The tech stack, on one slide
The four layers · which phase touches which

Plan-wide invariants · P0 if violated

Ten rules layered on top of AGENTS.md’s nine global invariants. Violating any of them is a P0 bug regardless of test status.

  1. TDD per task. Failing test → minimal implementation → green → commit.
  2. Commit per task. <type>(help|messages|time|screenshots): <summary>.
  3. Spec is canonical. Disagreements open questions, not silent fixes.
  4. No raw error codes in user-visible UI. /help/errors shows codes as URL fragments only; visible heading is entry.title.
  5. impeccable v3 UI gate. Every /help/* page passes /impeccable critique + /impeccable audit before close-out.
  6. Supabase call-boundary discipline. Already enforced for requireAdmin() via a structural meta-test.
  7. Catalog extension is additive. messageFor signature stays unchanged; type widens with three nullable fields.
  8. Catalog–master-spec alignment. Every admin-log-only code derived from master-spec §12.4 has all user-facing fields nulled.
  9. lib/time/now.ts is the only render-side clock. Every server-side new Date() reachable from a screenshot manifest route migrates here or carries a per-line waiver.
  10. §5.6 affordance matrix is the §9.0.1 retrofit contract. Every M3/M9/M10 component with an affordance carries its documented data-testid.
02 Why M10 has to finish first
AC-12.22 · Unconditional gate

M12 doesn’t begin until M10 closes out. No soft-start, even on phases B / C / D that don’t directly touch UI. Earlier plan drafts let those phases start early; that permission was removed in r2.

Phase F captures real screenshots of the dashboard, per-show panel, preview-as-crew banner, and onboarding wizard. Those surfaces are owned by M3, M9, and M10. If they’re still moving, every WebP we capture is wrong by the time it lands. Phase B also mutates the shared message catalog, setting dougFacing: null on entries currently rendered by M9’s AlertBanner — pre-M10 commits risk invalidating assumptions still being settled.

A meta-test is a test that walks the codebase to enforce a structural rule — “every X has a Y,” “no surface has both A and B.” Pre-declared at plan time per AGENTS.md.

04 Disagreement-loop preempts

These were debated through eight rounds of spec review. The plan cites each one so reviewers don’t relitigate.

/help/* is dynamic
Not statically prerendered — requireAdmin() runs Supabase queries per request.
Catalog extension is additive
Three new nullable fields. Existing callers compile unchanged.
Predicate
severity !== "info" AND dougFacing != null. No selfContainedAction flag.
Alignment lives in M12
Setting dougFacing: null on admin-log-only codes is M12’s responsibility, not a separate milestone.
Server clock pin
Request-scoped header (X-Screenshot-Frozen-Now), not an env var. Gated by ENABLE_TEST_AUTH + Bearer secret.
Fixture layout
Flat fixtures/shows/raw/<fixture>.md + pdf-only split __INFO.md.
Preview-as-crew = crew context
For the renderer, despite living on /admin/* URLs. No Learn more → in previewed content; only the sticky banner is admin-context.
/help/errors trailing CTA
"If this keeps happening, tell Eric →" — not a self-linking Learn more →.
