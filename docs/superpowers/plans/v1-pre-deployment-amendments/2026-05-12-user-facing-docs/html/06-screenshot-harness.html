PHASE F

Same data, same clock, same pixels — every single time.

Phase F builds the screenshot pipeline. Playwright signs in as a test admin, freezes both clocks (browser + server), disables animations, suppresses realtime, captures each documented surface in light and dark themes, encodes as WebP via sharp, and writes to disk. A CI gate fails the build if anything drifts.

Screenshots are the most fragile thing in any doc set — change one pixel, and every PR’s CI lights up with diffs that don’t mean anything. The fix is determinism: capture the same page from the same data with the same clock, animation, theme, and realtime state every single time. If anything’s out of pin, the harness throws.

A CI workflow runs the harness on every PR and fails the build if even one WebP byte differs from what’s committed. So either the capture is reproducible, or merging is blocked.

The twelve reproducibility preconditions

A screenshot is "deterministic" only if every external source of variation is pinned. Phase F enforces twelve.

PRE 1

ENABLE_TEST_AUTH = true

Env flag. Production never sets it. Throws on startup otherwise.

PRE 2

Browser clock

context.clock.install({ time }) — Playwright primitive.

PRE 3

Theme imposed

addInitScript sets data-theme before page load.

PRE 4

Animations off

CSS *, *::before, *::after { animation-duration: 0s !important }.

PRE 5

Server clock

X-Screenshot-Frozen-Now header from Phase C.

PRE 6

Realtime suppressed

window.WebSocket replaced with a no-op class.

PRE 7

Signed-in test admin

Via existing signInAs from tests/e2e/helpers/.

PRE 8

Font hinting off

Chromium args: --font-render-hinting=none — sub-pixel stays stable across machines.

PRE 9

TZ + locale + motion

America/New_York, en-US, reducedMotion: "reduce".

PRE 10

Quiescence wait

Optional selector wait + networkidle + 500ms stability.

PRE 11

Fixture-range validation

Frozen instant must lie within the fixture’s operational date range. Throws clearly if not.

PRE 12

Pinned encoder

sharp({ quality: 90, effort: 4, smartSubsample: true }). Version pinned in package.json.

The eleven tasks

F.1

Manifest definition

The single source of truth. (key, route, fixture, frozenClockInstant, viewport) per entry. Seeds with 4 entries; grows as content authoring needs.

scripts/help-screenshots.manifest.ts
F.2 · test #14

Fixture-range parser

Parses fixture INFO tab. Returns { earliest, latest }. Handles flat + pdf-only split. ISO + US dates.

F.3

Capture script

Playwright entry point. Enforces all 12 preconditions. Captures both themes per entry. Encodes via sharp.

F.4

Playwright projects

Setup-project pattern (real test, not globalSetup function). Webserver on port 3004 (avoid prod-runtime-flip 3003 collision). Adds screenshots-help-setup, screenshots-help, and help-docs projects.

F.5

CI drift gate

GitHub Actions workflow. On PR: run pnpm screenshot:help, then git diff --exit-code. Drifted bytes block merge.

F.6 · test #10

<picture> contract

For every manifest entry: assert <Screenshot name> renders <picture> with both light + dark sources at expected paths.

F.7 · meta-test #9

Manifest integrity

Stale entries (key but no WebP), orphan WebPs (file but no key), invalid fixtures — all caught here.

F.8 · test #8

Coverage

Walk every .mdx under app/help/; find every <Screenshot name>; assert the manifest contains an entry.

F.9 · test #18

E2E clock-pipeline proof

Capture same surface at two frozen instants 60+s apart. Assert the rendered HTML differs in the expected way — proves the clock pin actually flows through to render.

F.10

Convert placeholders

Each Phase E page that landed with a <ScreenshotPlaceholder> now points at its manifest entry. Manifest grows as needed.

F.11

Capture real WebPs

Run pnpm screenshot:help. Commit the WebPs under public/help/screenshots/. From here, every PR’s CI runs the same harness.
