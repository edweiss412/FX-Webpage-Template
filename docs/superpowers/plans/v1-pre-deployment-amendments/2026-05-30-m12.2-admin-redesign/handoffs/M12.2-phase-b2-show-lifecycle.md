## 6. Watchpoints from prior adversarial review (the 3 plan rounds — DO NOT RELITIGATE)

Settled contracts; the impl-diff reviewer is pre-loaded against re-opening them. Implement as written.

1. **Re-read authoritative state AFTER the advisory lock (R1 + R2 — the dominant vector).** Every mutating per-show RPC reads ONLY the immutable `drive_file_id` before `pg_advisory_xact_lock`, then re-reads mutable predicates (`archived`/`published`/finalize-ownership) in a SEPARATE select AFTER the lock. This applies to `archive_show`, `unarchive_show`, AND the DEF-1 guard on `rotate_show_share_token`/`reset_picker_epoch_atomic`. A pre-lock read is stale once a concurrent Archive commits while waiting — it causes token re-rotation / Live-token stranding. Each surface is pinned by a deterministic two-connection race test (`raceArchiveAgainst` in `_b2Helpers`). Do NOT "simplify" the guard back into the pre-lock `select drive_file_id`.
2. **`unarchive_show` early-returns (no-op) before ANY mutation when `archived=false` under the lock.** The `where archived=true` guard on the `shows` UPDATE alone does NOT protect the unconditional token-rotation/cleanup statements; a stale double-Unarchive on a Live row would rotate its active token. Pinned by already-Held + already-Live negative-regression tests.
3. **Live-schema fixture fidelity (R2).** `_b2Helpers.seedShow` does NOT insert `show_share_tokens` (the `shows_create_share_token_after_insert` trigger creates it — explicit insert = PK collision); it reads the trigger-created token. `pending_syncs` inserts MUST include `warning_summary` (NOT NULL, no default). The full `pending_syncs` NOT-NULL-no-default set: `drive_file_id`, `staged_modified_time`, `parse_result`, `source_kind`, `warning_summary`.
4. **DEF-4 silent-skip discipline (spec R-rounds).** Archived cron/push skips run BEFORE any Drive fetch and write ZERO `sync_log` rows — do NOT route through the normal prepared-skip path (`logSync` at `runScheduledCronSync.ts:~1896`; its `"skipped" in result` guard does NOT suppress `{outcome:"skipped"}`). In-lock authoritative re-read in `processOneFile_unlocked` after `assertShowLockHeld`. `listPostgresLiveShows` filtered `archived=false`; `markMissingShow_unlocked` archived-guarded.
5. **`requires_resync` is a durable boolean column, not a `last_sync_status` literal.** Set by Unarchive; cleared by clean sync (applied OR no-op) / Apply / Discard. It is the crash-safe Publish gate.
6. **Publish does NO synchronous Drive fetch.** Freshness = last-completed-sync; the catch-up sync runs on Unarchive. Publish is one self-locking RPC with an atomic gate (`requires_resync=false ∧ !archived ∧ !finalize-owned ∧ no live non-wizard scratch/suppressor`).
7. **`_archive_show_core`/`_publish_show_core` are private** (REVOKE from all roles incl. service_role); only the wrapper RPCs are `grant execute … to authenticated`. Pinned by the private-core-privilege test.
