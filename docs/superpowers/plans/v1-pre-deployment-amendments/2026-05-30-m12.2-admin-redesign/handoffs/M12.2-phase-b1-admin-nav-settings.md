## 6. Watchpoints from prior adversarial review (the 13 plan rounds — DO NOT RELITIGATE)

These are settled contracts. Each was the subject of one or more review rounds; the resolution is in the plan at the cited task. Implement them as written; the impl-diff reviewer is pre-loaded against re-opening them.

1. **No pass-only tasks (R1–R2).** Every task has a real red→green. Audit coverage lives inside the behavior task. Layout-dimensions (Task 9.1) and transition-audit (Task 9.2) expect a real first-run FAIL (Tailwind v4 collapse / un-wired compound transitions) + a component-fix step + negative-regression. prod-runtime-flip (Task 8.4) uses negative-regression as its red step.
2. **Fail-visible completeness (R3–R4).** See the **Fail-visible completeness matrix** in the plan's Meta-test inventory: EVERY infra read (alertCount, AlertBanner, fetchDriveConnectionHealth, getActiveWatchedFolder) is fail-visible on ALL THREE sub-paths — client-construction throw, returned `{error}`, thrown await — and EVERY write UI surface (AddAdminForm, RevokeRowButton, **ReAddRowButton**) renders the inline `ADMIN_EMAIL_WRITE_FAILED`. NotifBell `infra_error` → distinct degraded bell, never a clean no-badge. AlertBanner renders the degraded banner (never `null`) on ANY infra fault incl. construction throw. A positive/degraded bell must never route to an empty `/admin#alerts`.
3. **Invariant-9 destructuring + registry + grep-shape mechanics (R5–R8).** Read the plan's **`_metaInfraContract` grep-shape recognition contract** before writing any registered helper. Hard rules: (a) destructure `{ data, error }` (never bare `{ count, error }`); (b) a registered file MUST contain a grep-recognized supabase await — `await supabase.from(...)` or `await <builderVar>` where `const <builderVar> = supabase…` — so bind a local `const supabase = <client>` (an indirect `await resolvedClient.client.from(...)` is NOT recognized and fails the meta-test); (c) a pure wrapper with no direct supabase await (`fetchEmbeddedAdminEmails` → `await listAdminEmails()`) is NOT an `infraRegistry` row — it carries an inline `// not-subject-to-meta` comment + a behavioral test. **3 infraRegistry rows** (alertCount, fetchDriveConnectionHealth, getActiveWatchedFolder) + **1 wrapper exemption**.
4. **alertCount null-count integrity (R11).** `fetchUnresolvedAlertCount` returns `infra_error` unless `typeof count === "number"`. A null count with no error is NOT a clean zero.
5. **Route-render proof + layer-aware hook (R9–R10).** Task 2.0 adds a test-only `requireAdmin*({ layer: "layout" | "page" })` + header `x-test-force-infra-fail: <layer>` honored by BOTH `requireAdmin` and `requireAdminIdentity` (the existing `x-help-force-infra-fail` is requireAdmin-only and not layer-scoped — preserve it for `/help`). The layout gates with `layer:"layout"`; pages default `layer:"page"`. Task 2.3 forces `x-test-force-infra-fail: page` so the layout succeeds while a page gate throws, proving Next routes the fault to the page boundary.
6. **All admin `error.tsx` boundaries render `ADMIN_ROUTE_LOAD_FAILED` (R12).** B1 REPOINTS the existing `app/admin/settings/admins/error.tsx` from `ADMIN_EMAIL_LIST_FAILED` → `ADMIN_ROUTE_LOAD_FAILED`. After B1 routes the admin-list read through the typed in-section `fetchEmbeddedAdminEmails` wrapper, the boundary only catches route/session faults — NOT list-read failures. `ADMIN_EMAIL_LIST_FAILED` is an **in-section** state only (rendered by `AdministratorsSection` on `infra_error`). Client boundaries use a FIXED code (cannot read `err.code`).
7. **Drive-health worst-of-fleet via untruncatable counts (B1 spec §3.1; hardened in the spec's own 27 rounds).** The pill reflects the WORST active show via `head:true` count queries with WHERE predicates (NOT the capped 500-row render set). Worst-first precedence (tiers 1–7 + positive); status-based tiers (hard-failures 4a-c + `sync_unknown` 5) precede the age-based stale tiers. No-folder short-circuits BEFORE the `drive_watch_channels` read. Status-line copy derives from health/reason — NEVER a static "Connected" prefix on a Warn/infra state. Shared null-guarded `lastReadClause` on every branch. `attentionCount` = winning-reason predicate count, NOT the total fleet.
8. **Build-vs-runtime dev gate (B1 §3.3).** DevToolsRow keys off the artifact-time `DEV_PANEL_PRESENT` (`lib/admin/__generated__/devPanelPresent.ts`, committed `false`), NOT runtime `process.env.ADMIN_DEV_PANEL_ENABLED`. `scripts/with-admin-dev-flag.mjs` writes it at build + restores `false` on exit. 9. **§12.4 three-lockstep (5 new codes + 1 edit).** Each catalog change lands in ONE commit: master spec §12.4 prose (`docs/superpowers/specs/2026-04-30-fxav-crew-pages-v1.md`) + `pnpm gen:spec-codes` regen of `lib/messages/__generated__/spec-codes.ts` + the row in `lib/messages/catalog.ts`.
