- **ShowsTable title-track collapse — RESOLVED (2026-05-31, commit `65fb626`).** The incidental finding from the
  screenshots remediation: the dashboard collapsed every show title to a 0px `minmax(0,1fr)` track at ALL viewport
  widths (not a 720-810 band, as a first reading suggested). Root cause, verified in-browser: `app/admin/layout.tsx`
  wrapped all admin pages in `max-w-4xl` (896px) − padding = **832px** usable; the Dashboard's own `<main>` declares
  `max-w-5xl` (1024) but is nested inside that 832px wrapper, so 832 always won. The two-col split (320px inbox +
  gaps + ShowsTable's ~484px fixed tracks) needs ~940px and never got it, so the title `1fr` resolved to 0 and the
  Show/Dates headers overlapped. **A Dashboard breakpoint change alone could NOT fix it** (`main` is 832 at every
  viewport) — the original brief's `min-[960px]` prescription was unsatisfiable. Fix (orchestrator-approved):
  `layout.tsx` `max-w-4xl → max-w-6xl` (unlocks `main` to its `max-w-5xl`/1024 cap; the audit confirmed inner admin
  pages self-cap narrower, so per-show/settings do NOT widen — only the dashboard benefits) + `Dashboard.tsx`
  two-col split gated at `min-[1080px]` (was `min-[720px]`): stacks single-column below 1080 (full-width table →
  ample title), side-by-side at/above. **Measured in-browser:** title track = **154px at the 1080 activation**
  (narrowest two-col case) and **162px at 1280** — both above the 120px floor. New band-sweep test
  (`admin-layout-dimensions.spec.ts`, [720..1280]) asserts the resolved first grid track ≥120px + no row overflow +
  no header/title overlap (failed at every band pre-fix, `Received: 0`); the header↔row shared-track assertion was
  relaxed to tolerate the ~2px the `1fr` absorbs because each row carries a 1px border the header lacks (the four
  FIXED tracks still assert exact). Baselines `dashboard-overview` + `review-queues-empty-state` (light+dark)
  regenerated via amd64 Docker; `preview-as-crew-banner` byte-identical (mobile 390 unaffected). Both impeccable
  gates PASS (external); Codex impl-diff clean on the fix.

0px pre-fix at every band). Two per-show bugs from the Codex impl-diff pass also fixed: `be46c6e` (sync footer surfaced
color-only non-ok status → textual health label, WCAG 1.4.1) and `7961dda` (archived per-show page suppressed the
mutating `ReSyncButton`; server-side guard → DEF-3). All re-run gates green; baselines re-regenerated (layout widening
changed the 1280px render); real-CI 10 SUCCESS / 2 SKIP. **Gate-gap lesson:** height-equality layout assertions do not
catch horizontal track collapse — sweep the intermediate viewport band, not just one desktop + one mobile width.
