## Plan-wide invariants (re-stated — these are P0; see AGENTS.md + spec §13)

1. **TDD per task:** failing test → run-it-fails → minimal impl → run-it-passes → commit. Never impl before its test.
2. **Conventional commits**, one task per commit: `feat(notify):` / `test(notify):` / `feat(sync):` / `feat(db):` / `feat(admin):` / `docs(spec):` etc.
3. **At-least-once + Resend Idempotency-Key is the delivery contract (RD-9).** No claim-before-send / session-lock / terminal-state / persistent-outbox machinery. The 409-invalid reissue key uses a **per-submission nonce** (§3.2), never `attempt_count`.
4. **ONLY a Resend `200` records `sent`.** `409 concurrent` → `retry_later` (no write). `409 invalid` → reissue-to-deliver (never sent-without-200, never terminal-failed). `infra_error` → `failed`/retry.
5. **Maintenance ALWAYS runs** (toggle/provider-independent): `SYNC_STALLED` detect/resolve, recovery-resolution, `EMAIL_DELIVERY_FAILED` reconciliation. Only DELIVERY is gated.
6. **Occurrence-epoch keys are MICROSECOND-precision** `floor(extract(epoch from <ts>)*1e6)::bigint` — never second-granular, never `Date.parse(<Date>)` (truncates ms; compute in SQL or from ISO string).
7. **Config-valid = `RESEND_API_KEY` AND `EMAIL_FROM` AND a validated absolute site origin** (3-way). Every email href is ABSOLUTE via that origin (never the localhost fallback).
8. **No raw codes / no `<…>` placeholders / no em dashes** in any email body or toggle copy (invariants 5 + DESIGN.md §9). The placeholder guard runs on the pre-HTML plain-text copy.
9. **Supabase/DB call-boundary discipline (invariant 9) — EVERY new notify boundary registers (R7 plan-review):** every new helper that touches Supabase OR the postgres.js service-role client distinguishes returned-vs-thrown faults and, IN THE SAME GREEN COMMIT, EITHER adds a registry row to a notify infra-contract meta-test (Task 3.10) OR carries an inline `// not-subject-to-meta: <reason>`. This applies to ALL of: `activeRecipients` (2.5), the candidate queries (3.5), the delivery-loop ledger writes + recipient re-check (3.6), the `EMAIL_DELIVERY_FAILED` reconciliation (3.7), the digest builder queries (3.8), the orchestrator heartbeat/toggle reads (3.9), the stall detector + recovery-resolution DB calls (3.3/3.4/0.7), and the two getters (3.1). A returned/thrown DB fault must surface as a discriminable typed result (e.g. `{ kind:'infra_error' }`), never a silent `continue`/skipped send/stale alert. Task 3.10 pins the registry; do NOT let any boundary ship unregistered.
10. **No per-show advisory lock in the notify job** — it mutates only `admin_alerts` (idempotent upsert) + `email_deliveries`.

// `code: "X"` (or a status→code map literal elsewhere) must NOT satisfy the
// guard, else removing the real producer call while leaving any string literal
// behind would keep it green (the exact pre-B3 "cataloged code, no producer"
// regression). The producer is `(...).upsertAdminAlert({ showId, code: "X", ...})`
// so the code property sits within ~160 chars after an upsertAdminAlert( call.
const PRODUCER_FILES = [
  "lib/sync/runScheduledCronSync.ts",
  "lib/sync/runManualSyncForShow.ts",
] as const;
describe("SYNC_PROBLEM_CODES producer existence (anti-drift, per-path, call-bound)", () => {
  for (const file of PRODUCER_FILES) {
    const src = readFileSync(file, "utf8");
    for (const code of SYNC_PROBLEM_CODES) {
      test(`${file} has an upsertAdminAlert producer for ${code}`, () => {
        // upsertAdminAlert( … code: "<CODE>" … ) within the same call object
        const re = new RegExp(`upsertAdminAlert\\([\\s\\S]{0,200}?code:\\s*["'\`]${code}["'\`]`);
        expect(re.test(src), `no upsertAdminAlert({ code: "${code}" }) producer call in ${file}`).toBe(true);
      });
    }

export type SiteOriginResult = { ok: true; origin: string } | { ok: false };
/** Validated canonical origin for ABSOLUTE email links (§5.4). NEVER the localhost dev fallback. */
export function resolveSiteOrigin(raw: string | undefined = process.env.NEXT_PUBLIC_SITE_ORIGIN): SiteOriginResult {
  if (!raw || !raw.trim()) return { ok: false };
