Imagine a Lyft receipt page, but for working a touring AV show. You open your phone backstage and see only the things you need: which hotel you're in, what time call is, which truck you're riding, what gear is on your pull sheet today, who to call if the LED wall goes dark. M3 Admin proving rig

The throwaway upload form

/admin/dev is an internal page Eric uses to prove the parser plus the Phase-1 sync logic actually work end-to-end on real fixtures, before any of the production cron infrastructure exists. It does a real write — it has to, AC-3.2 and AC-3.3 explicitly assert that rows land in pending_syncs and pending_ingestions. A pure preview wouldn't satisfy the contract.

The trick: it writes into an isolated dev_* schema that mirrors public. The migrations apply twice — once to public for production, once to dev for the panel — so a fixture upload can never corrupt real shows rows. A "Reset dev schema" button truncates everything in dev between runs.

Build-time gate · server-only flag

An earlier draft used NEXT_PUBLIC_ENABLE_ADMIN_DEV_PANEL — but the NEXT_PUBLIC_ prefix inlines the value into the client bundle and can be mutated at runtime. The corrected design uses a server-only ADMIN_DEV_PANEL_ENABLED flag, read once during Server Component render. A dual-Playwright-build test proves the prod artifact returns 404 even for an admin; the dev artifact returns 403 for non-admins. The dev panel must never reach production builds, regardless of auth state.

M4 The crew page

What the crew member actually sees

Open /show/<slug>?crew=<crewMemberId> on a phone. A Header with the show title. A Right Now card that always knows what's happening — pre-travel, travel-in, set, show day 2, strike, post-show. Then a 2-column grid of tiles, each rendering one narrow slice of the show data. A Footer. That's the whole page.

Every tile reads from a single Server Component data fetcher, getShowForViewer(showId, viewer). That helper is the only place role flags get derived — the page never trusts caller-supplied roles, never accepts a ?role= query param, and re-reads crew_members.role_flags from the database on every request. M5 will replace the identity-only mock with real auth; the contract here is built so that swap is a one-line change.

Invariant · identity-only mock

The M4 page accepts ?crew=<id> for identity and ?as=admin for admin preview. ?role= is explicitly ignored even if present — a regression test (tests/e2e/role-spoof.spec.ts) asserts that ?role=lead cannot unlock the FinancialsTile when the bound crew row's role_flags don't include LEAD. getShowForViewer binds its lookup on both id AND show_id, closing the cross-show financial-leak vector.

The Right Now card · state machine

The card at the top of the page never says "Welcome" or "Today's date." It says what is happening right now, computed from the show's date list, the viewer's day-restriction, and the current wall-clock time in the venue's timezone. Twelve states. Each transition has an explicit animation treatment in the audit matrix.

M4 · selectRightNowState — pure function, 12 states, table-ordered precedence
Transition matrix · 12 states · 66 pairs

Per the global CLAUDE.md "Transition inventory" rule, every state-transition pair gets an explicit treatment: crossfade-body for date rollovers (container holds min-h-[X] to preserve card height), morph-to-last-good for any → unknown, instant for user-initiated changes, unreachable with a regression guard for impossible paths (e.g., post_show → pre_travel). Compound transitions — sync-error mid-rollover, role demotion mid-state — get six dedicated integration tests.

Gotcha · Tailwind v4 does not default .flex to align-items: stretch

This project's Tailwind v4 setup means every fixed-dimension parent containing flex/grid children needs explicit items-stretch + h-full (or an equivalent inline style). Jest + jsdom is not sufficient to catch the failure mode — jsdom doesn't compute real layout, so a layout-collapse bug passes unit tests. Task 4.13 writes the real-browser assertion: getBoundingClientRect() on every documented data-testid, asserting child.dimension === parent.dimension within a 0.5px tolerance. Five invariants from §8.4, exhaustively covered.

The visibility contract, in code

The single shared SCOPE_TILE_VISIBILITY_RULE imported by every consumer — the scope tiles, the transition audit, the §17.1 acceptance criteria. No inline restatement anywhere in the codebase. 

// lib/visibility/scopeTiles.ts — single source of truth.
import type { RoleFlag } from '@/lib/parser/types';

export function audioScopeVisible(flags: RoleFlag[]): boolean {
  return flags.includes('A1') || flags.includes('A2') || flags.includes('LEAD');
}

export function videoScopeVisible(flags: RoleFlag[]): boolean {
  return flags.includes('V1') || flags.includes('LEAD');
}

export function lightingScopeVisible(flags: RoleFlag[]): boolean {
  // §8.1 amended 2026-05-13 — LEAD now reads-in to Lighting scope,
  // symmetric with Audio/Video. See spec Amendment §8.1-A1.
  return flags.includes('L1') || flags.includes('LEAD');
}

export function financialsVisible(flags: RoleFlag[], isAdmin: boolean): boolean {
  return isAdmin || flags.includes('LEAD');
}

M3·M4 invariants — never violated past this milestone

  1. UI quality gate. Every UI surface ships only after /impeccable critique AND /impeccable audit pass on the affected diff, with HIGH/CRITICAL findings either fixed or explicitly deferred via DEFERRED.md.
  2. Identity-only viewer. getShowForViewer never accepts role_flags as a parameter. Role is re-derived from crew_members.role_flags on every call. Static-analysis regression test pins the signature.
  3. Show-bound lookup. Crew lookup binds on both id AND show_id. A LEAD crew row from a different show cannot inherit financial visibility on this show.
  4. Canonical scope-tile rule. One predicate module. No viewerRole === 'LEAD' checks anywhere in tile components.
  5. No raw Drive URLs in crew DOM. Opening-reel renders URL-stripped status text at M4. Inline <video> defers to M7 Task 7.6 once the asset-proxy route exists.
  6. Real-browser layout assertions. Tailwind v4's non-default flex behavior makes jsdom insufficient. Every dimensional invariant gets a Playwright assertion.
