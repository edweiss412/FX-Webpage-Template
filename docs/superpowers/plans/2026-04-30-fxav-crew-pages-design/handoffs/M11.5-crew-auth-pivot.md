## 6. Watchpoints from prior adversarial review

Surface-by-surface failure modes recorded across 45 spec rounds + 36 plan rounds:

- **`select_identity_atomic` timestamp source** — pre-R41-R23, `now()` returned transaction-start; under lock contention, claim_oauth_identity in a concurrent transaction could stamp earlier than Mallory's observed_at_millis → impersonation. Fix: `v_claim_at := clock_timestamp()` AFTER all locks acquired. **Watchpoint**: any new RPC that compares a session/cookie timestamp to a DB-stamped time MUST use `clock_timestamp()` AFTER lock acquisition.

- **PostgreSQL banker's rounding (R41-R30 HIGH)** — `::bigint` cast rounds half-to-even. mint_safe_t_millis and resolver must both use `floor(extract(epoch from ...) * 1000)::bigint` consistently OR resolver-fails-its-own-mint regressions appear. **Watchpoint**: any new ms timestamp derivation MUST use `floor()` cast, NOT bare `::bigint`.

- **Cookie decoder unix-seconds cap (P-R15 CRITICAL)** — pre-fix cap of `2_000_000_000` (year 2033 unix-seconds) rejected every ~1.7e12 ms timestamp. **Watchpoint**: any new cookie field treated as a timestamp MUST be validated against `MAX_SAFE_T_MILLIS = Number.MAX_SAFE_INTEGER`.

- **Service-role bypass of `is_admin()` (P-R17 Fix-2 + P-R30 Fix-1)** — service-role clients have NO JWT; `auth.jwt()` returns NULL; `is_admin()` / `auth_email_canonical()` always return null. **Watchpoint**: any new RPC gated by `is_admin()` MUST be called from cookie-bound client; `GRANT EXECUTE` should NOT include service_role (structural defense — silent NULL return is the failure mode).

- **Shared-device identity leak vector (P-R27/P-R30 CRITICAL)** — pre-fix, Bob's Google session + Alice's stale picker cookie would resolve as Alice on BOTH page route (step 4(e) fall-through) AND API consumers (cookie-only resolver had no session-vs-cookie consistency check). **Watchpoint**: any new code path that consumes the picker cookie MUST either (a) gate on Google-session-matches-cookie OR (b) be on the page route and route through resolveShowPageAccess.
