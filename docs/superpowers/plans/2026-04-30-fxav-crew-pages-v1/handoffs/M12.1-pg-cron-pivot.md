## §6 — Watchpoints

Watch for during self-review + adversarial review:

- **Schedule equivalence drift.** The 7 `cron.schedule()` calls in T3 MUST match `vercel.json`'s schedules at HEAD `ac752d9` byte-for-byte. Numeric sweep per AGENTS.md spec-self-review additions. The T4 meta-test pins this contractually.
- **Bootstrap signing-key cron preservation.** The pre-existing cron at `supabase/migrations/20260504000001_bootstrap_nonces_signing_key.sql:36` is untouched; T3's `cron.unschedule()` is scoped to `jobname like 'fxav_cron_%'`. If a reviewer claims the bootstrap cron is at risk, cite the LIKE clause + the T4 meta-test assertion 4 (non-fxav cron count > 0).
- **Vault secret rotation lifecycle.** Value is populated post-migration (Phase 0.A.5 for validation env; M13 launch task for prod env). Default placeholder forces 401 from /api/cron/* if not populated. If a reviewer asks "what if the secret is wrong?" — that's the fail-loud design intent, not a bug.
- **`app.fxav_vercel_url` GUC ergonomics.** Single operational pre-req before T3 migration applies. Alternative designs (env-var-templated migration; per-row constants) were considered and rejected for simplicity. If a reviewer pushes back on the GUC choice, surface this rationale (don't relitigate).
- **No handler code changes.** Verify `app/api/cron/*/route.ts` files are NOT in any commit's diff. Any reviewer claim that handler code "needs" modification is incorrect; cite `app/api/cron/_auth.ts:3-12` + the 7 handler verification points in spec §1.2.

## §7 — Do-not-relitigate

EXPLICITLY DO NOT RELITIGATE the following, ratified contracts (cite when reviewer pushes back):

- **pg_cron over GitHub Actions / Cloudflare / Upstash / cron-job.org.** Ratified in spec §1.1. Comparison table includes reliability, granularity, vendor surface, free-tier cost. The trade-off basis is documented; do not re-derive.
- **Vault over GUC for `CRON_SECRET` storage.** Ratified in spec §2.3. Rationale: encrypted at rest, rotation-friendly, isolated from `pg_settings` visibility. Reviewer arguing for GUC simplicity should be answered with "Vault chosen per spec §2.3 rationale; defer to BACKLOG if you want to revisit."
- **`vercel.json` crons block is REMOVED (not retained-with-comment, not retained-but-ignored).** Ratified in spec §2.1 + plan T1. Hobby tier rejects deployments declaring crons exceeding daily even when those crons would not fire. Removal is the only deploy-passing path.
- **No route-handler code changes.** Ratified by live-code citation in spec §1.2 + §4. All 7 handlers already accept `Authorization: Bearer $CRON_SECRET` via `_auth.ts`.
- **Sub-amendment dir vs M12 R71+ amendment.** Ratified by orchestrator-user decision 2026-05-26. M12 R70 APPROVE stays intact; M12.1 has its own loop.
- **Stop gates A+B+C (M12 amendment loop) do NOT carry forward.** M12.1 uses standard adversarial-review pattern per AGENTS.md routing.
