- **Schedule equivalence drift.** The 7 `cron.schedule()` calls in T3 MUST match `vercel.json`'s schedules at HEAD `ac752d9` byte-for-byte. Numeric sweep per AGENTS.md spec-self-review additions. The T4 meta-test pins this contractually.

If a reviewer claims the bootstrap cron is at risk, cite the LIKE clause + the T4 meta-test assertion 4 (non-fxav cron count > 0).
- **Vault secret rotation lifecycle.** Value is populated post-migration (Phase 0.A.5 for validation env; M13 launch task for prod env). Default placeholder forces 401 from /api/cron/* if not populated. If a reviewer asks "what if the secret is wrong?" — that's the fail-loud design intent, not a bug.
- **`app.fxav_vercel_url` GUC ergonomics.** Single operational pre-req before T3 migration applies. Alternative designs (env-var-templated migration; per-row constants) were considered and rejected for simplicity. If a reviewer pushes back on the GUC choice, surface this rationale (don't relitigate).
- **No handler code changes.** Verify `app/api/cron/*/route.ts` files are NOT in any commit's diff. Any reviewer claim that handler code "needs" modification is incorrect; cite `app/api/cron/_auth.ts:3-12` + the 7 handler verification points in spec §1.2.
