### 3.6 Malformed stored-item validation + value bounding (writer, fail-safe)

Because stored items reaching the writer (Â§3.5) are guarded only for `id`/`invariant` (`reviewPayloadGuards.ts:53,74`), the structured writer MUST NOT assume any other field is present or well-typed. The prior generic writer never dereferenced `field`/`crew_name`/`prior`/`next`, so a corrupt/forward-schema stored item was harmless; the new writer dereferences them and must **fail safe, never `undefined`, never throw**. Per-entry runtime validation (at the writer, before building each entry) â€” **a validation failure skips the entry entirely, never producing a placeholder/fake entry** (an all-malformed sync then writes the structured Unavailable-marker row, Â§3 step 3):

- **MI-8:** `item.field` must be one of `po | proposal | invoice | invoiceNotes`; otherwise **skip** (no partial/`undefined` label). Note-only entry â€” no value dereference.
- **MI-8c:** `item.mode` must be one of `collapse | ambiguous_format | halved | case_dropped`; otherwise **skip**.
- **MI-9:** `item.crew_name` must be a non-empty string (after trim) and `item.prior_flags`/`item.new_flags` must each be an **array of strings**; otherwise **skip** (no partial/`undefined` label, no fake role entry). Each flag string is coerced trim-aware and the set rendered via the flag-join (Â§3.4b); an empty array renders `"(none)"`. A well-formed MI-9 item produces its Fromâ†’To entry and does **not** touch `omittedCount`; only a malformed one is skipped and counted.
- **MI-8b:** requires `prior` and `next` keys that are each `string | null`; then normalize both trim-aware (`x = typeof x === "string" && x.trim() !== "" ? x.trim() : "(none)"`). **Skip if the shape is invalid OR the two normalized values are equal** â€” this drops a corrupt `{ id, invariant:"MI-8b" }` (both â†’ `"(none)"`, equal â†’ skipped, **no fake `(none)â†’(none)` row**) and mirrors the live fire condition `priorCoi !== nextCoi` (`invariants.ts:460`). Otherwise emit Fromâ†’To with the normalized values (never a blank cell).
- **Universal (trim-aware):** every string placed in an entry (`label`/`from`/`to`/`note`) is coerced through `typeof x === "string" && x.trim() !== "" ? x.trim() : <sentinel>` (`<sentinel>` = `"(none)"` for MI-8b values) so the stored JSON never contains `undefined`, `null`, `""`, or whitespace-only in a taken branch â€” upholding the "no blank cell" guarantee (Â§7). (`label` values are writer-controlled constants and always non-empty.)

**Omitted items â†’ incompleteness marker (Codex R12/R15).** The writer tracks `omittedCount` = malformed MI-8/8b/8c/9 items skipped by per-entry validation (Â§3.6). A partial list must never be presented as complete on a whole-row-Accept feed:
- **â‰¥1 valid entry AND `omittedCount > 0`** â†’ after the valid entries, append **one bounded marker entry** `{ label: "Other changes", note: "<omittedCount> other field change(s) on this sync â€” details unavailable", from: null, to: null }`. - **0 valid entries** â†’ the **row-emission predicate `hasFieldFamilyItem` (Â§3 step 0)** decides: if any field-family item was present (i.e. every field-family item was malformed) â†’ the **structured Unavailable-marker row** (Â§3 step 3: `after_image.fieldChanges = [ one "Unavailable" marker ]` + an omission-encoding `summary`, never `after_image = null`); else no row.

The marker/fallback is an **in-band data signal** (not telemetry): the writer runs inside the sync advisory-lock txn, so per invariant 10 it emits no `log.*`/`logAdminOutcome` â€” the marker entry (and the generic-fallback row) carry the "something was omitted" signal in the persisted row itself. **Value bounding (storage + render):** each stored **`label`/`from`/`to`/`note`** string is **truncated at the writer** to **120 chars** with a trailing `â€¦` when longer. The user-sourced strings are MI-8b COI status (`from`/`to`) and MI-9's `crew_name` (in the `label`) + role-flag join (`from`/`to`, Â§3.4b); the other labels/notes are writer constants (`"PO number"`, `"COI status"`, `"Pull sheet â€¦"`, `"Role â€” â€¦"` prefix, the marker note) already short.
