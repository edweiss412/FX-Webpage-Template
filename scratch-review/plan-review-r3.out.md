- [HIGH] `FinalizeBlockerModal` is specified as a self-gating component that is always rendered by `FinalizeStatusRegion`, while also using `useDialogFocus`. If placed before the `if (!mounted || !active) return null`, the idle render consumes the effect; if placed after it, it violates Rules of Hooks. The plan needs an outer gate plus an inner mounted dialog component. See `docs/superpowers/plans/2026-07-17-wizard-blocker-modal.md:105`, `:110`, `:111`, `:270`; `lib/a11y/dialogFocus.ts:45`.
